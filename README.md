# Companion

A personal AI video companion, built on the Tavus Conversational Video Interface.
You create one companion, give it a name, and have live video calls with it. It
remembers what you talked about last time, offers games when things get quiet,
notices if you seem low or excited, and speaks with emotional TTS instead of a
flat assistant voice.

## How it works

- **Backend** — Node.js + Express. Holds the Tavus and Anthropic API keys
  server-side and proxies every call; the browser never talks to either API
  directly.
- **API keys** — entered through the app itself (Settings, top right), not
  `.env`. Stored locally in `data/keys.json` (gitignored). This means the
  actual key values only ever go from your browser to your own local server —
  never typed into a chat, a committed file, or anywhere else.
- **Memory** — a JSON file (`data/companion.json`). After every call, the
  transcript is summarized by Claude (`claude-sonnet-4-6`) into an updated
  running summary, a capped list of atomic facts, and a warm one-line greeting
  shown the next time the app opens.
- **Frontend** — a single page (`public/index.html` + `public/app.js`). Embeds
  the live call as a Tavus-hosted iframe; no Daily.js integration.
- **Persona config** — `persona-config.js` + `lib/tavus-persona.js` define the
  companion's system prompt, perception layer, TTS emotion settings, and the
  trivia tool, and apply them to the Tavus persona via `PATCH /v2/pals/{id}`.

## Setup

```bash
npm install
cp .env.example .env
npm start
```

Open `http://localhost:3000`. Click **Settings** (top right) and paste in
your Tavus and Anthropic API keys — they're saved locally and used from then
on. Then create your companion and you're ready to call. If you try to call
before keys are set, the app shows a toast and opens Settings for you.

### Environment variables (`.env`)

Only the pieces that aren't API keys live here:

| Variable | Required | What it's for |
|---|---|---|
| `PORT` | No (default `3000`) | Local server port |
| `PUBLIC_BASE_URL` | Only for trivia | A public HTTPS URL that reaches this server — see below |
| `TAVUS_TOOL_WEBHOOK_SECRET` | Only for trivia | HMAC secret for verifying tool-call webhooks; auto-generated by the upgrade script if left blank |

`TAVUS_API_KEY` / `ANTHROPIC_API_KEY` in `.env` are no longer read on startup
— if you had them there from before this feature existed, they're
automatically migrated into `data/keys.json` the first time the server
starts, and you can safely remove them from `.env` afterward.

## Features

- **Persistent memory** — summary, facts, and a personalized greeting carry
  across calls. Facts are capped at 40 (oldest dropped first).
- **Orphan-call recovery** — if a call drops without a clean hangup (closed
  tab, network blip, hit the 30-minute cap), the next page load or call
  automatically finalizes it and still saves what was learned.
- **Games** — the companion offers trivia, would-you-rather, two truths and a
  lie, or build-a-story when conversation lulls. Would-you-rather / two
  truths / build-a-story are fully conversational. Trivia calls a real
  question from [Open Trivia DB](https://opentdb.com/) via a tool the LLM
  invokes mid-call, and can show it as a tappable Interactive Card (Tavus
  Magic Canvas) instead of only speaking it.
- **Perception** — the persona watches for signs the person seems low/tired
  or excited/energized (visually and by voice) and is instructed to respond
  like a friend noticing, not a diagnostic tool.
- **TTS emotion** — Cartesia voice engine with `tts_emotion_control` enabled,
  so tone shifts are audible. Full effect depends on the replica's model
  generation — see Known limitations.

## Trivia requires a public URL

Tavus delivers tool calls and Interactive Card answers to your server via
webhook, so it needs to reach you — `localhost` doesn't work. Without
`PUBLIC_BASE_URL` set, everything else works normally; trivia just isn't
registered.

```bash
ngrok http 3000
# copy the https://*.ngrok-free.app URL into .env as PUBLIC_BASE_URL
node scripts/upgrade-persona.js
```

Re-run the upgrade script any time `PUBLIC_BASE_URL` changes.

## Scripts

| Command | What it does |
|---|---|
| `npm start` | Run the server |
| `npm run dev` | Run with `node --watch` for auto-restart on file changes |
| `node scripts/upgrade-persona.js` | Retrofit trivia/perception/TTS-emotion/system-prompt onto the existing companion's Tavus persona. New companions get all of this automatically at creation — this is only for updating one that already exists. |

## Dev preview mode

Screens that normally require a live call can be previewed without spending
call credits:

- `?preview=recap` — the "what I'll remember" screen with sample facts
- `?preview=ending` — the "saving what I learned" loading screen
- `?preview=call` — the live-call layout (no real video)

## Project structure

```
server.js              Express app — all API routes and webhooks
store.js                JSON-file-backed memory store
persona-config.js       System prompt + perception/TTS layers + trivia tool definition
lib/
  keystore.js             Reads/writes data/keys.json — the only place API keys touch disk
  tavus-client.js         Thin fetch wrapper for the Tavus API
  tavus-persona.js        Applies persona-config.js to a real Tavus persona
  trivia.js               Open Trivia DB fetcher (topic matching, decoding, shuffling)
scripts/
  upgrade-persona.js     One-off: retrofit an existing persona with new capabilities
public/
  index.html             The whole UI (single page, four views, settings modal, toast)
  app.js                  View logic, API calls, call timer, settings/keys handling
data/
  companion.json          Local memory store (gitignored)
  keys.json               Local API keys (gitignored, never committed)
```

## Known limitations

- **Trivia's webhook needs a public URL.** See above.
- **TTS emotion depends on the replica.** Cartesia's emotion control is
  tuned for Phoenix-4 faces; the default stock replica here is Phoenix-3, so
  expect a partial effect until/unless you switch to a Phoenix-4 replica.
- **No mid-call context injection.** Tavus's live context/interrupt events
  (e.g. reinforcing "game mode" mid-call) are sent client-side via a Daily.js
  call object. This app embeds Tavus's hosted iframe instead, which doesn't
  expose that — implementing it would mean replacing the iframe with a
  Daily.js integration.
- **Single companion.** The app supports exactly one companion at a time,
  stored locally in `data/companion.json`.
